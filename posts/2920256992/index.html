SpringMVC

MVC：

  • 模型（Model）+ 视图（View）+ 控制器（Control），是一种软件设计规范
  • MVC主要作用：降低视图与业务逻辑间的双向耦合
  • 是一种架构模式
  • 最典型的MVC模式 ：JavaBean+JSP+Servlet

Spring MVC Framework是一个开放源代码Java平台，可提供全面的基础结构支持，以非常轻松，快速地开发基于Java的强大Web应用程序。

Spring Web MVC框架提供了一个模型-视图-控制器架构和现成的组件，可用于开发灵活且松散耦合的Web应用程序。MVC模式导致分离应用程序的不同方面（输入逻辑，业务逻辑和UI逻辑），同时在这些元素之间提供松散的耦合。

  • 该模型封装了应用程序数据，通常，它们将由POJO组成。

  • 该视图 负责呈现模型数据，并在一般情况下，它会生成HTML输出，客户端的浏览器可以解释。

  • 该控制器负责处理用户请求和建立适当的模型，并将其传递到用于呈现该视图。

DispatcherServlet

Spring Web模型视图控制器（MVC）框架是围绕处理所有HTTP请求和响应的DispatcherServlet设计的。下图显示了Spring Web MVC DispatcherServlet的请求处理工作流。

以下是与对DispatcherServlet的传入HTTP请求相对应的事件序列

  • 收到HTTP请求后，DispatcherServlet咨询HandlerMapping来调用适当的Controller。

  • 控制器接收请求并基于使用的GET或POST方法调用适当的服务方法。服务方法将基于定义的业务逻辑设置模型数据，并将视图名称返回给DispatcherServlet。

  • DispatcherServlet将获得ViewResolver的帮助，以获取请求的已定义视图。

  • 视图完成后，DispatcherServlet将模型数据传递到视图，该视图最终在浏览器上呈现。

上述所有组件，即HandlerMapping，Controller和ViewResolver都是WebApplicationContext的一部分，它是对纯ApplicationContext的扩展，具有Web应用程序所需的一些额外功能。

SpringMVC运行原理

流程：

graph LR
A(用户) --1.发送请求-->B(DispatcherServlet:前端控制器)
B--2.委托请求到处理器-->C(Handle:处理器/页面控制器)
C --3.调用业务对象-->D(Model:模型)
D --4.返回模型数据 -->C
C --5.返回ModelAndView-->B
B--6.Model渲染视图-->E(View:视图)
E--7.将视图返回前端控制器 -->B
B--8.产生响应-->A
graph LR
A(user)--1 Request-->B(dispatcherServlet)
B--2-->C(HandlerMapping)
C--3-->D(HandlerExecution:具体的Handler)
D--4-->B
B--5-->E(HandlerAdapter)
E--6-->F(Controller)
F-.调用业务.->Z(Service Business Logic)
Z-->X(Repository Data Access)
X-->V(Mysql)
F-.->G(Model)
F--7-->E
E--8-->B
B--9-->H(ViewResolver)
H--10-->B
B--11-->I(view)
H-.->I
I-.->G
I--12 Response -->A

接收请求参数

@Controller
public class UserController {
    /** utl参数与方法参数名一致的情况下，可以直接获取传递的参数值
     * 访问http://localhost:8080/test1?name=fan
     * 参数名即使一致的情况下，最好在方法的参数名添加@RequestParam注解
     * @param name
     * @param model
     * @return test.jsp页面
     */
    @GetMapping("/test1")
    public String test1(String name,String id, Model model){
        System.out.println("获取到的参数值："+name );
        System.out.println("id值："+id);
        model.addAttribute("msg","参数值name:"+name+";id:"+id);
        return "test";
    }

    /**
     * url参数名与方法的参数名不一致 ，方法参数名添加@RequestParam("url参数名")
     * 访问：http://localhost:8080/test2?userid=1
     * @param id
     * @param model
     * @return
     */
    @GetMapping("/test2")
    public String test2(@RequestParam("userid") int id, Model model){
        model.addAttribute("msg","test2()方法；参数值："+id);
        return "test";
    }

    /**
     * 前端传递的是一个对象；例如：User 对象
     * 匹配User对象的字段名，前提：传递的参数名与字段名须一致
     * 访问：http://localhost:8080/test3?name=fan&id=1 可直接获取
     * @param user
     * @param model
     * @return
     */
    @GetMapping("/test3")
    public String test3(User user,Model model){
        model.addAttribute("msg","test3()方法;"+user.toString());
        return "test";
    }
}

SpringMVC 拦截器

  • Spring MVC的拦截器类似于 Servlet的 过滤器Filter，用于处理器进行预处理和后处理

  • 拦截器是 AOP 思想的具体实现

    拦截器与过滤器的区别：

  • 拦截器是 Spring MVC 框架自有 的，使用SpringMVC框架才能使用 ，过滤器是Servlet的规范的一部分 ，java web工程都可用

  • 拦截器 只会拦截访问的控制器方法，若访问的是 路径文件 是不会拦截；过滤器在url-pattern中配置/* 后，可对所有访问的资源进行过滤

自定义拦截器

实现自定义拦截器必须 实现 HandlerInterceptor 接口

public class Myinterceptor implements HandlerInterceptor {

    /**
     * return true：执行下一个拦截器 return false：则不执行下一个拦截器
     */
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("========处理前");
        return true;
    }

   /*
   *后两个的作用可以追加日志
   */
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("========处理后");
    }

    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("========清理");
    }
}

登录拦截器

public class LoginInterceptor implements HandlerInterceptor {
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        /**
         * 放行：
         * 若 session：userLoginInfo 不为空，return true；
         * 登录页面页需要放行 ： URI中包含login
         *
         */
        HttpSession session = request.getSession();
        if(session.getAttribute("userLoginInfo") !=null){
            return true;
        }
        if(request.getRequestURI().contains("Login")||request.getRequestURI().contains("login")){
            return true;
        }
        /**
         * 不放行：  转发至登录页面
         */
        request.getRequestDispatcher("/WEB-INF/jsp/login.jsp").forward(request,response);
        return false;
    }
}

SpringMVC 文件上传和下载

文件上传 一般 会使用 Apache 下的 Commons FileUpload组件 ，需要在Servlet 完成 servlet版本应在3. 0以上

SpringMVC 上下文中默认没有装配MultipartResolver ，若需要使用Spring 的文件上传功能，则需要在上下文中配置 MultipartResolver。

  1. form表单需要 post请求，并配置 enctype=“multipart/form-data”
  2. 上下文配置 MultipartResolver
  3. 导入 commons-fileupload 包 和 javax.servlet-api

扩展： enctype属性 ：

  • application/x-www=form-urlencoded:默认方式，只处理form表单中 域的 value 值，会将表单域中的值 处理成 URL 编码格式
  • multipart/form-data： 以二进制流的编码格式处理表单数据，会把文件域中指定文件的内容封装到请求参数中，不会对字符编码 适用文件上传
  • text/plain: 除了把 空格 转换成 + 符号，其它字符都不做编码处理，适用直接通过表单发送邮件

SpringMVC 为文件上传提供直接的支持，实现方式：MultipartResolver 即插即用

SpringMVC 使用 Commons FileUpload 技术实现 MultipartResolver实现类 ：CommonsMultipartResolver

@RestController
public class FileController {
    @RequestMapping("/upload")
    public String upload(@RequestParam("file")CommonsMultipartFile file, HttpServletRequest servletRequest){
        String path = servletRequest.getServletContext().getRealPath("/upload");
        File filePath  = new File(path);
        if(!filePath.exists()){
            filePath.mkdir();
        }
        try {
            file.transferTo(new File(filePath+"/"+file.getOriginalFilename()));
        } catch (IOException e) {
            e.printStackTrace();
        }
        return  "ok";
    }

    @RequestMapping("/download")
    public String  download(HttpServletResponse response,HttpServletRequest request){

        //要下载的文件地址
        String path = request.getServletContext().getRealPath("/upload");
        // 下载的文件名 若有空格 则  空格会转化为+
        String fileName  = "新建文本文档 (2).txt";
        InputStream in = null ;
        OutputStream out = null;
        try {
            //设置response
            response.reset();
            response.setCharacterEncoding("utf-8");
            response.setContentType("multipart/form-data");
            //设置响应头
            response.setHeader("Content-Disposition", "attachment;fileName=" + URLEncoder.encode(fileName, "utf-8"));

            File file = new File(path, fileName);
            in = new FileInputStream(file);
            out = response.getOutputStream();

            byte[] bytes = new byte[1024];
            int len = 0;
            //执行写出操作
            while ((len = in.read(bytes)) != -1) {
                out.write(bytes, 0, len);
                out.flush();
            }
        }catch (Exception e){
            e.printStackTrace();
        }finally {
            try {
                in.close();
                out.close();
            }catch (Exception e){
                e.printStackTrace();
            }

        }
    return "ok";
    }
}
