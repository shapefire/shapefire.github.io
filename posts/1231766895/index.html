RestFul风格

REST

  • REST全称是Representational State Transfer，表述性状态转移
  • 基于HTTP、URI、XML、JSON等标准和协议
  • 支持轻量级、跨平台、跨语言

轻量级：代码不被侵入，跟包大小无关，耦合性越低，越轻量 。反例：Struts中每一个Action都要继承核心控制器

REST主要原则

  • 对网络上所有的资源都有一个资源标志符(URI)
  • 对资源的操作不会改变标识符
  • 同一资源有多种表现形式（xml、json）
  • 所有操作都是无状态的（Stateless)

URI(Universal Resource Identifier ）:统一资源标志符：在某一规则下能把一个资源独一无二地标识出来

URL(Universal Resource Locator ):统一资源定位符 ：提供找到该资源的路径

URL是URI的子集

无状态性：使得客户端和服务器端不必保存对方的详细信息，服务器只需要处理当前的请求，不需了解请求的历史。可以更容易的释放资源，让服务器利用Pool技术来提高稳定性和性能。

RESTful概述

  • RESTful是一种常见的REST应用，是遵循REST风格的web服务，REST式的web服务是一种ROA（面向资源的架构）

    web服务交互方案：

    1.REST

    2.SOAP(Simple Object Access protocol):简单对象访问协议是交换数据的一种协议规范，是一种轻量的、简单的、基于XML的协议，它被设计成在WEB上交换结构化的和固化的信息

    3.RPC(Remote Procedure Call Protocol):远程过程调用协议，它是一种通过网络从远程计算机程序上请求服务，而不需要了解底层网络技术的协议

  • 基于RestFul风格设计的软件可以更简洁，更有层次，更易于实现缓存机制

资源与URI

  • 任何事物，只要有被引用到的必要，它就是一个资源
  • 资源可以是实体，，也可以是抽象的
  • 要让一个资源可以被识别，需要有个唯一标识，在Web中这个唯一标识就是URI(Uniform Resource Identifier)

例如：查询user的id为xx的信息

REST的url(uri): http://…/user/1

非REST的url：http://…/user?id=1

统一资源接口

  • RESTful架构遵循统一接口原则，统一接口包含了一组受限的预定义的操作，不论什么样的资源，都是通过使用相同的接口进行资源的访问。

  • 按照HTTP方法的语义来暴露资源，那么接口将会拥有安全性和幂等性的特性；例如GET和HEAD请求都是安全的， 无论请求多少次，都不会改变服务器状态。而GET、HEAD、PUT和DELETE请求都是幂等的

安全性：对该REST接口访问，不会使服务器端资源的状态发生改变。

幂等性: 无论对资源操作多少次， 结果总是一样的，后面的请求并不会产生比第一次更多的影响,得到的资源状态是一样的

资源操作

http 方法 资源操作 安全性 幂等性
GET SELECT 是 是
POST INSERT 否 否
PUT UPDATE 否 是
DELETE DELETE 否 是

传统方式操作资源 ：通过不同参数来实现不同的效果 方法单一：GET,POST

传统URL请求格式：

URL 方式
http://127.0.0.1/user/query?id =1 查询，GET
http://127.0.0.1/user/save 添加，POST
http://127.0.0.1/user/update 更新，POST
http://127.0.0.1/user/delete?id =1 删除，POST

RESTful操作资源：通过不同的请求方式来实现不同的效果： 例如请求URL相同，但是实现的功能不同

RESTful请求格式

URL 方式
http://127.0.0.1/user/1 查询，GET
http://127.0.0.1/user 添加，POST
http://127.0.0.1/user 更新，PUT
http://127.0.0.1/user/1 删除，DELETE

使用测试：

测试环境：SpringMVC

  • Controller类
@Controller
public class RestFulController {
    /**
     * 传统操作资源方式：
     * 访问 url:http://localhost:8080/add?a=1&b=10
     * @param a
     * @param b
     * @param model
     * @return test.jsp
     */
    @RequestMapping("/add")
    public String traditionalOperating(int a, int b, Model model){
        int result = a+b;
        model.addAttribute("msg","结果为"+result);
        return "test";
    }

    /**
     * 使用RestFul风格操作资源方式 默认：GET
     * 访问uri：http://localhost:8080/add/1/10
     * 方法参数前需要添加 @PathVariable注解 让方法参数的值对应绑定到一个URI模板变量上
     * @param a
     * @param b
     * @param model
     * @return test.jsp
     */
    /**
     *  @GetMapping("add/{a}/{b}")
     */
    @RequestMapping("add/{a}/{b}")
    public String restFulOperatingTest1(@PathVariable int a, @PathVariable int b,Model model){
        int result = a+b;
        model.addAttribute("msg","结果为"+result);
        return "test";
    }

    /**
     * 使用RestFul风格操作资源方式 通过指定的请求方式实现
     * 在 @RequestMapping 注解中 添加 method属性：
     * 属性值有： GET,HEAD,POST,PUT,PATCH,DELETE,OPTIONS,TRACE;
     * 例如指定请求方式为Delete：method = RequestMethod.DELETE
     * @param id
     * @param model
     * @return test.jsp
     */
    @RequestMapping(value = "/delete/{id}",method = RequestMethod.DELETE)
    /**
    *上面注解可用@DeleteMapping("/delete/{id}"),其他属性值类似：GET请求则为@GetMapping
    */
    public String restFulOperatingTest2(@PathVariable int id,Model model){
        /**
         * 执行删除用户id业务层操作
         */
        return "test";
    }
}

SpringMVC 发送PUT，Delete请求问题

  1. 在浏览器端的 Form 表单中只支持 GET、POST 请求，不支持 PUT、DELETE 请求

    解决：

    • web.xml下增加名称为hiddenHttpMethodFilter的过滤器

      <filter>
        <filter-name>hiddenHttpMethodFilter</filter-name>
        <filter-class>org.springframework.web.filter.HiddenHttpMethodFilter</filter-class>
    </filter>
    <filter-mapping>
        <filter-name>hiddenHttpMethodFilter</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

      HiddenHttpMethodFilter 过滤器的原理是获取一个 name 为 _method 的属性，获取其值，根据其值决定转化为 PUT 还是 DELETE 请求，所以需要在表单中设置一个 name 值为 _method 的隐藏域元素。

      PUT请求方式 若该请求已在form表单method=“post”中不需要借助JavaScrippt

      Delete 请求 ，没有在from表单中，需要添加隐藏域的form表单并借助js

      <form action="/book/" method="post" id="del_From">
    <input type="hidden" name="_method" value="DELETE">
</form>
<script type="text/javascript">
    $(function () {
        //为超链接为class=‘del’点击动作添加行为函数
        $(".del").click(function () {
            //获取class为delete的超链接的地址
          // var href = $(this).attr("href");
            //为隐藏域的action赋值并提交
            $("#del_From").attr("action", this.href).submit();
            return false;//该语句不可缺少 将超链接的默认行为取消
        })
    })
</script>
