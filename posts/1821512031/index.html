IOC(inversion of control): 对象有Spring容器创建，对象属性是Spring容器来设置

控制的内容：指谁来控制对象的创建；传统的应用程序是由程序本身控制，使用Spring后，由Spring来创建对象

反转：正转指程序创建对象，反转指程序本身不去创建对象，而变为被动接收的对象

IOC —- 也称为依赖注入（Denpendency injection ）

静态代理：

  • 静态代理的角色分析：
    • 抽象角色—–一般使用接口或者抽象类来实现
    • 真实角色—–被代理的角色
    • 代理角色—–代理真实角色——代理真实角色后一般会做一些附属操作
    • 客户 ——使用代理角色来进行一些操作

​ Rent.class (抽象角色)

    public interface Rent {
        public void rent();
    }

​ Host.class(真实角色)

    public class Host implements Rent {
        public void rent() {
            System.out.println("房屋出租");
        }
    }

​ Proxy.class(代理角色)

    public class Proxy  implements Rent{
        private Host host;
        public Proxy() {    
        }
        public Proxy(Host host) {
            this.host = host;
        }
        //租
        public void rent() {
            seeHouse();
            fare();
            host.rent();
        }

        //看
        public void seeHouse() {
            System.out.println("带房客看房");
        }

        //收中介费
        public void fare() {
            System.out.println("收取中介费");
        }
    }

​ Client.class(客户)

    public class Client{
        public static void main(String[] args) {
            Host host =new Host();
            Proxy proxy = new Proxy(host);
            proxy.rent();
        }
    }
  • 使用静态代理的好处：
    • 使得真实角色处理的业务更加纯粹，不再去关注一些公众的事情
    • 公共的业务由代理完成—-实现业务的分工
    • 公共业务发生扩展时变得更加集中于方便
  • 缺点
    • 类多了 —-多了代理类。工作量变大，开发效率降低了

AOP：面向切面编程 aspect oriented programming

在不改变原有代码的情况下添加功能

aop在Spring的作用：

  • 提供声明式服务（声明式事务）

  • 允许用户实现自定义切面

名词解释：

  • 关注点：增加的某个业务。如安全，异常处理，日志，缓存等；
  • 切面：一个关注点的模块化
  • 通知：在切面的某个特定的连接点上执行的动作
  • 织入：把切面连接到其他的应用程序类型或对象上，并创建一个被通知的对象

  • 使用注解实现AOP

    //Log.class
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
@Aspect
public class Log {
    @Before("execution(* com.zcf.service.impl.*.*(..))")
    public void before() {
        System.out.println("----方法执行前----");
    }
    @After("execution(* com.zcf.service.impl.*.*(..))")
    public void after() {
        System.out.println("----方法执行后----");
    }
    @Around("execution(* com.zcf.service.impl.*.*(..))")
    public Object around(ProceedingJoinPoint jp) throws Throwable {
        System.out.println("环绕前");
        System.out.println("签名：" + jp.getSignature());
        //执行目标方法
        Object result = jp.proceed();
        System.out.println("环绕后");
        return result;

    }
}

  • mybatis的别名配置元素设置

    • 使用typeAlias 

      <typeAliases>
         <typeAlias alias="User"   type="com.**.entity.User"/> 
</typeAliases>

      此种方式的缺点就是每增加一个实体类都需要加上对应的类配置

    • 使用package 

      <typeAliases>
        <package name="com.**.entity"/>
</typeAliases>

      此种方式是扫描包下的类，使用标签，表示扫描该包名下的所有类（除了接口和匿名内部类），如果类名上有注解，则使用注解指定的名称作为别名，如果没有则使用类名首字母小写作为别名，如com.XX.entity.User这个类如果没有设置@Alias注解，则此时会被关联到user这个别名上。

事务

事务时正确执行一系列的操作或动作，使得数据库从一种转态转换成另一种状态，且保证操作全部成功，或全部失败

原则：确保数据库操作的ACID特性

  1. 原子性：不可分割性
  2. 一致性
  3. 隔离性 ：在事务提交之前，它可能的结果不应显示个任何其它事务
  4. 持久性：事务正确提交后，其结果将永久保存在数据库中

事务类型：

  1. JDBC事务：用Connection对象控制，包括手动和自动模式
  2. JTA事务：与实现无关的，与协议无关的API
  3. 容器事务：应用服务器提供的，且大多是基于JA完成（通常基于JNDI的，相当复杂的API实现）

​ 差异：JDBC事务局限于一个数据库连接内

​ JTA事务：功能强大，课跨越多个数据库或多Dao，使用比较复杂

​ 容器事务：主要指J2EE应用服务器提供的事务管理，局限于EJB

事务的读取类型：

  1. 脏读： 事务没提交，提前读取
  2. 不可重复读：两次读取的数据不一致
  3. 幻读：事务不是独立执行时发生的一种非预期现象

事务的隔离级别：定义一个事务可能受其他并发事务影响的程度

事务的传播特性（行为）：当事务方法被被另一个事务方法调用是，必须指定事务如何传播 有7种

事务是否只读：利用数据库事务的只读属性，进行特定优化处理

​ 注意：不同的数据库厂商支持不同，例如Oracle的“readOnly”不起作用，不影响增删修查，而mysql则其作用

                            http://www.springframework.org/schema/tx
                        https://www.springframework.org/schema/tx/spring-tx.xsd">
   <!-- 配置数据源 -->
    <bean id= "dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
        <property name="driverClassName" value="com.mysql.jdbc.Driver"></property>
        <property name="url" value="jdbc:mysql://localhost:3306/studdb?serverTimezone=UTC" />
        <property name ="username" value="root" />
        <property name="password" value="123456" />
    </bean>
    <!-- 配置sqlsessionFactory -->
    <bean id ="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource"></property>
        <property name="configLocation" value="classpath:spring_mybatis.xml"></property>
    </bean>
    <bean id="sqlSessionTemplate" class="org.mybatis.spring.SqlSessionTemplate">
        <constructor-arg index="0" ref="sqlSessionFactory" />
    </bean>

    <!-- 声明式事务 -->
    <!-- 配置事务管理器 -->
    <bean id ="txManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <property name="dataSource" ref="dataSource"></property>
    </bean>
    <!-- 配置事务通知 -->
    <tx:advice id ="txAdvice" transaction-manager="txManager">
        <tx:attributes>
            <!-- 配置哪些方法使用什么样的事务，配置事务的传播特性 -->
            <tx:method name="add" propagation="REQUIRED"/>
            <tx:method name="delete" propagation="REQUIRED"/>
            <tx:method name="select" propagation="REQUIRED"/>
            <tx:method name="update" propagation="REQUIRED"/>
            <tx:method name="remove*" propagation="REQUIRED"/>
            <tx:method name="get" read-only="true"/>
            <tx:method name="*" propagation="REQUIRED"/>
        </tx:attributes>
    </tx:advice>

    <aop:config>
        <aop:pointcut expression="execution(* com.zcf.dao.impl.*.*(..))" id="pointcut"/>
        <aop:advisor advice-ref="txAdvice" pointcut-ref="pointcut"/>
    </aop:config>
